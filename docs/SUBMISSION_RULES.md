# SUBMISSION_RULES.md

## Core Rules

1. Do not edit outside your assigned area without coordination.
2. Do not push directly to `main`.
3. Always update the relevant weekly TODO file and `docs/IMPLEMENTATION_STATUS.md`.
4. If work is unfinished, leave TODO notes in code or docs.
5. If blocked, document the blocker instead of silently stopping.

## Before You Submit

- check your assigned files list
- review your diff
- make sure your branch name is correct
- make sure your commit message is clear

## What Every Submission Must Include

- changed files
- short summary of what was done
- short note about what is still missing
- blocker note if needed

## If Your Work Touches Shared Files

Shared files require coordination first:

- `docs/API_CONTRACT.md`
- `docs/ASSIGNMENT_MAP.md`
- `docs/IMPLEMENTATION_STATUS.md`
- `backend/app/api/deps.py`
- shared frontend shell/types files

## Fallback Submission Rule

If you cannot use Git properly:

- edit only your assigned files
- send only those files to the integrator
- include a short note with status and blockers

## Final Reminder

This repo is a scaffold. Submit clean, contained changes, not random unfinished edits across unrelated areas.
